# AI Rules for ZenPulse Flutter App

Ты — senior Flutter разработчик мобильных приложений.

## Общие правила

- Пиши чистый и читаемый Flutter код
- Избегай лишних зависимостей
- Используй стандартные Flutter Material виджеты
- Разделяй UI на небольшие переиспользуемые виджеты

## Архитектура проекта

Структура проекта:

lib/
  screens/
  widgets/
  services/
  models/

Правила:
- screens — экраны приложения
- widgets — переиспользуемые UI компоненты
- services — бизнес логика
- models — модели данных

## Mobile UX правила

- Каждый экран должен быть обёрнут в SafeArea
- Контент должен быть scrollable
- Используй ListView для списков
- Избегай фиксированных высот (height: 400 и т.п.)
- Используй Flexible и Expanded для адаптивности
- UI должен корректно работать на маленьких экранах (iPhone SE)

## UI правила

- Современный минималистичный дизайн
- Большие кликабельные кнопки
- Скруглённые карточки
- Отступы между элементами
- Используй мягкие цвета (под стиль meditation apps)

## Логика приложения

В приложении есть флаг:

bool isSubscribed = false

Правила:

Если isSubscribed == false:
- некоторые медитации должны быть заблокированы
- карточки должны быть серыми
- показывать иконку замка
- при нажатии открывать Paywall

Если isSubscribed == true:
- пользователь видит весь контент

## AI генерация текста

AI аффирмации генерируются через мок-сервис.

Используй:
Future.delayed()

для имитации ответа AI.

## Код

- код должен компилироваться
- избегай длинных методов
- разделяй UI на виджеты
